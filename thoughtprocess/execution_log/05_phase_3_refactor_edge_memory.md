## Step 1: Refactor DevopsScene Edge Memory (RED Phase)

### Intention
We need the `DevopsScene` to persistently store the mathematical `VMobject` paths it generates during `render_topology()`. If it simply draws the paths and loses the variable references, we cannot apply a Manim `MoveAlongPath` animation to them later. 

The intention is to implement an internal dictionary on `DevopsScene`: `self.rendered_edges: dict[tuple[str, str], VMobject] = {}`.
When `topology.edges` are processed, the resultant `VMobject` line generated by the `OrthogonalRouter` must be stored using the `(source_id, target_id)` tuple as the dictionary key.

### Actions
1.  **Red Phase:** Write `test_devops_scene_stores_edge_memory_IMMUTABLE` in `tests/test_core.py`. Instantiate the scene, run `render_topology()` on a generic 2-node graph, and `assert len(scene.rendered_edges) == 1`. It will fail because `rendered_edges` doesn't exist.
2.  **Green Phase:** Edit `manim_devops/core.py`. Add the dictionary to `DevopsScene.__init__()` and populate it inside the `render_topology()` edge loop.
